# 宝塔站点 SSH

宝塔站点 SSH 是一个 Electron 桌面工具，用于给宝塔面板中的站点创建独立 SSH 登录账号。它适合需要把某个站点目录交给开发者、运维人员或自动化任务访问，但又不想直接开放服务器 `root` 账号的场景。

## 功能

- 管理多台宝塔服务器配置
- 使用 `root` 密码或 `root` 私钥连接服务器
- 校验 SSH 连通性、主机指纹和 `root` 权限
- 读取宝塔站点列表，必要时回退到 `/www/wwwroot` 扫描
- 为站点生成本地 `ed25519` 密钥并写入远程 `authorized_keys`
- 创建或更新站点专用 Linux 用户
- 使用 ACL 给专用用户授权站点目录，不修改目录属主
- 站点路径位于其他网站子目录中时，会给上级目录追加最小穿透权限
- 写入本机 `~/.ssh/config`
- 测试站点账号在站点目录中的新建、编辑、删除权限
- 禁用站点 SSH，并移除本机对应 SSH 配置

## 环境要求

- Node.js 18 及以上
- macOS 桌面环境
- 可通过 SSH 访问的宝塔服务器
- 远程账号必须是 `root`，或具备完整 `root` 权限
- 远程系统需要可用的 `useradd`、`passwd`、`getent`、`find`
- 启用站点 SSH 需要远程已安装 `setfacl`
- 读取宝塔数据库时优先使用远程 `python3` 和 `sqlite3`；不可用时会回退到目录扫描

## 使用方法

1. 打开应用后，点击左侧顶部的加号添加服务器。
2. 填写服务器名称、地址、端口和登录用户。登录用户必须具备完整 `root` 权限。
3. 选择认证方式：
   - 密码：填写服务器 `root` 密码
   - 私钥：选择本机可读取的 `root` 私钥，可按需填写私钥口令
4. 点击“保存服务器”。应用会先校验 SSH 连接、SSH 主机指纹和 `root` 权限，校验通过后才会保存配置。
5. 如果是首次连接该服务器，应用会显示 SSH 主机指纹。请先通过带外渠道核对指纹，再点击“信任指纹并保存”。
6. 如果后续服务器重装、SSH 服务重建或主机密钥轮换，应用会提示“SSH 主机指纹不匹配”。确认变更属实后，再点击“更新指纹并保存”。
7. 选择服务器后，点击“读取站点”加载宝塔站点。
8. 在站点列表中点击“启用”，确认远程用户名和本地密钥保存路径。
9. 点击“创建并写入配置”。完成后即可使用应用显示的 SSH 信息连接站点账号。
10. 启用后可点击“测试”，检查专用账号是否能在站点目录中创建、编辑和删除文件或目录。
11. 不再需要站点 SSH 时，点击“禁用”。应用会锁定远程用户，并移除本机由本工具写入的 SSH 配置块。

## 连接方式

启用站点 SSH 后，应用会写入本机 `~/.ssh/config`。通常可以使用以下方式连接：

```bash
ssh <远程用户名>@<服务器地址>
```

如果启用时生成了 SSH 别名，也可以使用：

```bash
ssh <别名>
```

本地私钥默认保存在：

```text
~/.ssh/bt-ssh/<服务器地址>/<远程用户名>_ed25519
```

启用时可以改成其他保存路径，但应填写私钥路径，不要填写 `.pub` 公钥路径。

## 权限测试说明

站点测试只关注站点 SSH 账号对站点目录的文件读写能力，不测试 Nginx、Apache、HTTPS、入口文件或宝塔配置。

测试范围：

- 站点根目录
- 最多 8 个现有子目录
- 目录深度不超过 4 层

会跳过一些常见大目录或缓存目录：

- `.git`
- `node_modules`
- `vendor`
- `cache`

每个被测试目录都会执行以下动作：

1. 新建目录
2. 重命名目录
3. 删除目录
4. 新建文件
5. 追加编辑文件
6. 删除文件

所有测试文件和目录都使用临时名称，测试结束后会清理。

## 安全说明

- 服务器凭据只保存在本机应用数据目录中，并通过 Electron `safeStorage` 加密
- 首次连接服务器时不会自动信任未知 SSH 主机
- 已保存服务器如果返回不同的 SSH 主机指纹，应用会阻止连接
- 本工具不会把服务器凭据上传到外部服务
- 本工具不会修改站点目录属主，不会对站点目录执行 `chown`
- 启用时会递归追加 ACL；站点文件较多时首次启用可能耗时较长
- 如果站点路径本身位于其他目录下，启用时会给上级目录追加仅可进入的 ACL，不开放读取列表或写入权限
- 启用时会锁定专用用户的密码登录，只保留公钥登录
- 删除服务器配置不会自动删除本地 SSH 密钥，也不会自动删除远程用户

## 注意事项

- 不要把生产服务器的 `root` 密码或私钥交给不可信设备
- 首次连接或指纹变更时，请务必通过控制台、运维平台或已有可信通道核对 SSH 主机指纹
- 如果远程用户名已经存在，且不像本工具创建的站点账号，启用会失败；请换一个远程用户名
- `~/.ssh/config` 中由本工具管理的配置块带有 `bt-ssh-gui` 标记，手动修改这些块可能影响应用识别启用状态
- 如果本地私钥文件已经被删除，重新读取站点后将不会继续显示对应“本地秘钥”路径
