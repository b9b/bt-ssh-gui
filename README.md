# 宝塔站点 SSH

宝塔站点 SSH 是一个 macOS 桌面工具，用来给宝塔面板中的单个站点创建独立 SSH 登录账号。它适合把某个站点目录交给开发者、运维人员或自动化任务访问，同时避免直接开放服务器 `root` 账号。

## 功能概览

- 管理多台宝塔服务器
- 使用 `root` 密码或 `root` 私钥连接服务器
- 校验 SSH 连通性、主机指纹和 `root` 权限
- 读取宝塔站点列表，必要时回退到 `/www/wwwroot` 目录扫描
- 为单个站点创建独立 Linux 用户
- 生成本地 `ed25519` 密钥，并把公钥写入远程 `authorized_keys`
- 使用 ACL 授权站点目录，不修改站点目录属主
- 写入本机 `~/.ssh/config`，方便直接 `ssh <别名>` 连接
- 测试站点账号在站点目录中的创建、编辑、删除权限
- 禁用站点 SSH，并清理本工具写入的本地 SSH 配置和远程 ACL

## 环境要求

- macOS 桌面环境
- 可通过 SSH 访问的宝塔服务器
- 远程登录账号必须是 `root`（`UID=0`）
- 远程系统需要可用的 `useradd`、`passwd`、`getent`、`find`
- 启用站点 SSH 需要远程已安装 `setfacl`
- 禁用时如果远程缺少 `setfacl`，会跳过 ACL 清理，只执行能完成的锁定用户和本机配置清理
- 权限测试需要远程可用 `su`、`mktemp`、`chown`、`chmod`
- 读取宝塔数据库时优先使用远程 `python3` 和 `sqlite3`；不可用时会回退到目录扫描

## 快速使用

1. 打开应用，点击左侧顶部的加号添加服务器。
2. 填写服务器名称、地址、端口和登录方式。
3. 保存服务器。应用会先校验 SSH 连接、主机指纹和 `root` 权限。
4. 首次连接服务器时，应用会显示 SSH 主机指纹。请通过控制台、运维平台或已有可信通道核对后再信任。
5. 选择服务器，点击“读取站点”。
6. 在站点列表中点击“启用”，确认远程用户名、SSH 别名和本地私钥路径。
7. 点击“创建并写入配置”。
8. 启用成功后，可以使用应用显示的命令连接站点账号，也可以点击“测试”检查目录权限。
9. 不再需要时，点击“禁用”。

## 启用时会执行什么

启用入口是站点列表中的“启用”按钮，以及弹窗中的“创建并写入配置”。启用动作分为本机操作和服务器端操作。

### 本机操作

1. 根据站点名称、站点 ID 或用户输入生成远程 Linux 用户名。
2. 根据服务器名称、服务器地址和站点名称生成 SSH 别名。
3. 确定本地私钥路径。默认路径类似：

   ```text
   ~/.ssh/bt-ssh/<服务器地址>/<远程用户名>_ed25519
   ```

4. 如果私钥不存在，执行 `ssh-keygen` 生成无密码的 `ed25519` 密钥。
5. 设置本地私钥权限为 `600`。
6. 如果 `.pub` 公钥文件不存在，会从私钥重新导出公钥，并设置公钥权限为 `644`。
7. 读取本地公钥内容，准备写入服务器。
8. 等服务器端操作成功后，写入或更新本机 `~/.ssh/config`：
   - 写入一个按服务器地址和端口分组的 endpoint 配置块
   - 写入一个站点 SSH 别名配置块
   - 配置 `HostName`、`Port`、`User`、`IdentityFile`
   - 配置 `IdentitiesOnly yes`
   - 配置块带有 `bt-ssh-gui` 标记，方便后续识别和清理
9. 设置 `~/.ssh/config` 权限为 `600`。
10. 在本机应用数据中记录该站点已启用，以及对应的 SSH 别名、远程用户名和本地私钥路径。

### 服务器端操作

1. 检查远程是否存在 `setfacl`。不存在时启用会失败，并提示先安装 `acl`。
2. 检查远程是否存在 `useradd`。不存在时启用会失败。
3. 检查站点路径是否存在且是目录。不存在时启用会失败。
4. 如果远程用户已存在，应用会判断它是否像本工具管理的站点账号：
   - 用户家目录下的 `site` 符号链接指向当前站点路径；或
   - 用户的 `authorized_keys` 中包含 `bt-ssh-gui` 标记
5. 如果远程用户已存在但不像本工具管理的账号，启用会失败，避免误改已有系统用户。
6. 如果远程用户不存在，执行 `useradd -m -s /bin/bash <用户名>` 创建用户。
7. 执行 `passwd -l <用户名>` 锁定该用户的密码登录。
8. 创建或修正用户家目录下的 `.ssh` 目录：
   - 目录权限为 `700`
   - 属主和属组为该站点用户
9. 用本机公钥覆盖写入远程用户的 `~/.ssh/authorized_keys`。
10. 设置远程 `authorized_keys`：
    - 属主和属组为该站点用户
    - 权限为 `600`
11. 在远程用户家目录创建或更新 `site` 符号链接，指向真实站点目录。
12. 对站点路径的所有上级目录追加最小穿透 ACL：

    ```text
    u:<远程用户名>:--x
    ```

    这允许站点用户进入嵌套路径，但不允许列出上级目录内容，也不允许写入上级目录。

13. 对站点目录内的现有文件和目录递归追加 ACL：

    ```text
    u:<远程用户名>:rwx
    ```

    `.user.ini` 会被跳过。

14. 对站点目录内的所有目录追加默认 ACL：

    ```text
    default:u:<远程用户名>:rwx
    ```

    这样后续新建文件或目录会继承站点用户权限。

15. 如果服务器上存在 `www` 用户，也会给 `www` 用户补充当前 ACL 和默认 ACL：

    ```text
    u:www:rwx
    default:u:www:rwx
    ```

    这样可以降低站点 SSH 用户和 Web 服务用户之间的写入权限冲突。

### 启用时不会做什么

- 不会修改站点目录属主
- 不会对站点目录执行 `chown`
- 不会删除站点文件
- 不会修改 Nginx、Apache、PHP、HTTPS 或宝塔站点配置
- 不会自动给非当前站点授权

## 禁用时会执行什么

禁用入口是站点列表中的“禁用”按钮。禁用同样分为本机操作和服务器端操作。

### 本机操作

1. 从本机 `~/.ssh/config` 中删除该站点 SSH 别名配置块。
2. 从本机 `~/.ssh/config` 的 endpoint 配置块中移除该站点使用的 `IdentityFile`。
3. 如果该服务器 endpoint 配置块已经没有其他本工具管理的密钥，会一并删除该 endpoint 配置块。
4. 重新写入 `~/.ssh/config`，并保持权限为 `600`。
5. 从本机应用数据中移除该站点的启用状态。

### 服务器端操作

1. 如果远程用户存在，执行 `passwd -l <用户名>` 锁定用户。
2. 如果远程存在 `setfacl`，且站点目录仍然存在，则递归移除站点用户在站点目录内的 ACL：

   ```text
   u:<远程用户名>
   ```

   `.user.ini` 会被跳过。

3. 对站点目录内的所有目录移除站点用户的默认 ACL：

   ```text
   default:u:<远程用户名>
   ```

4. 从站点路径的所有上级目录移除站点用户的穿透 ACL：

   ```text
   u:<远程用户名>
   ```

5. 返回“站点 SSH 已禁用”。

### 禁用时不会做什么

- 不会删除本地私钥或公钥文件
- 不会删除远程 Linux 用户
- 不会删除远程用户家目录
- 不会删除远程用户的 `authorized_keys`
- 不会删除远程用户家目录里的 `site` 符号链接
- 不会回滚启用时给 `www` 用户追加的 ACL
- 不会修改站点目录属主
- 不会删除站点文件

## 连接方式

启用成功后可以直接使用远程用户名连接：

```bash
ssh <远程用户名>@<服务器地址>
```

也可以使用应用写入的 SSH 别名：

```bash
ssh <别名>
```

连接后，远程用户家目录中的 `site` 符号链接会指向真实站点目录。

## 权限测试说明

站点测试只检查站点 SSH 账号对站点目录的文件读写能力，不测试 Nginx、Apache、HTTPS、入口文件或宝塔配置。

测试前会先确认：

- 站点目录存在
- 远程站点用户存在
- 远程站点用户可以进入站点路径

测试范围：

- 站点根目录
- 最多 8 个现有子目录
- 子目录深度不超过 4 层

会跳过以下常见大目录或缓存目录：

- `.git`
- `node_modules`
- `vendor`
- `cache`

每个被测试目录都会执行以下动作：

1. 新建目录
2. 重命名目录
3. 删除目录
4. 新建文件
5. 追加编辑文件
6. 删除文件

所有测试文件和目录都使用临时名称，测试结束后会清理。

## 安全与注意事项

- 服务器凭据只保存在本机应用数据目录中，并通过 Electron `safeStorage` 加密
- 本工具不会把服务器凭据上传到外部服务
- 首次连接服务器时不会自动信任未知 SSH 主机
- 已保存服务器如果返回不同的 SSH 主机指纹，应用会阻止连接
- 指纹变更可能来自服务器重装、SSH 主机密钥轮换，也可能来自中间人风险；请核对后再更新信任
- 启用会递归追加 ACL；站点文件较多时首次启用可能耗时较长
- 如果远程用户名已经存在，且不像本工具创建的站点账号，启用会失败；请换一个远程用户名
- `~/.ssh/config` 中由本工具管理的配置块带有 `bt-ssh-gui` 标记，手动修改这些块可能影响应用识别启用状态
- 删除服务器配置不会自动删除本地 SSH 密钥，也不会自动删除远程用户
- 如果本地私钥文件已经被删除，重新读取站点后将不会继续显示对应“本地秘钥”路径
